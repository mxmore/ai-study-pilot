# 安全与合规基线

## 数据安全
- 所有敏感数据（用户身份、学习记录）均存储在受控的 VPC 中，数据库启用静态加密。
- 访问凭证通过 Vault/Secrets Manager 管理，禁止硬编码在仓库中。
- MinIO/对象存储启用分级访问策略，Worker 仅拥有必要的读写权限。

## 隐私合规
- 对用户行为日志进行匿名化处理，仅保留必要的学习画像。
- 提供导出与删除数据的 API，满足 GDPR/中国个人信息保护法要求。
- AI 生成内容需记录来源，便于追踪与纠错。

## 模型安全
- 对外部大模型调用设置速率限制与审计日志。
- 引入 Prompt 注入检测：对输入进行白名单过滤与上下文裁剪。
- 质量评估 Agent 必须运行在隔离环境，防止互相影响。

## DevSecOps
- 提供 Schema 校验脚本（见 `scripts/validate_question.py`）作为 CI 检查之一。
- 后端单元测试需覆盖权限控制与输入验证，发布前执行 SAST/DAST。
- 采用基础依赖检查（如 `npm audit`, `pip-audit`）追踪安全漏洞。
